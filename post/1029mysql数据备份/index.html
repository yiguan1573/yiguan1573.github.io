DataX mysql 项目

mysql数据备份

Posted by     "亿观" on Sunday, May 8, 2022

前言

  • 公司很久以前的一个库查询太慢，需要做主从复制和读写分离，但是这个库数据量大几百G而且不能停机，所以需要将数据热备份到从库后再做其他操作

DataX

  • 开始想到DataX，它是阿里开源的一个异构数据源离线同步工具，可以在不同的数据库之间同步
  • 但是后来发现它没有建库建表操作，所以放弃
  • 使用详情翻看我的博客

xtrabackup

  • 依赖包下载，下载地址

  • Xtrabakcup下载地址

  • 安装步骤

    [root@k8s-master01 ~]# ls
libev-4.04-2.el6.x86_64.rpm  Percona-XtraBackup-2.4.21-r5988af5-el7-x86_64-bundle.tar
[root@k8s-master01 ~]# yum localinstall libev-4.04-2.el6.x86_64.rpm -y
[root@k8s-master01 ~]# tar xf Percona-XtraBackup-2.4.21-r5988af5-el7-x86_64-bundle.tar 	# 解压完之后出出现三个rpm
[root@k8s-master01 ~]# yum install percona-xtrabackup-24-2.4.21-1.el7.x86_64.rpm -y

  • 修改配置文件/etc/my.cnf，保证[mysqld]模块存在参数datadir=/var/lib/mysql（指向数据目录），因为xtrbackup是根据/etc/my.cnf配置文件来获取需要备份的文件

  • 查看是否安装成功

    xtrabackup --version
innobackupex --version

  • 全量备份

    innobackupex --user=root --password=****** /home/mysqlbackup/  2>> /home/mysqlbackup/bakcup.log
#第一个路径保存备份数据，第二个路径保存备份输出信息

  • 在恢复前，需要使用–apply-log参数合并数据文件，确保数据的一致性，后面跟上备份的数据目录

    innobackupex --apply-log ./2022-05-25_19-22-32

  • 数据还原(需要先关闭数据库，且需要原来存数据的目录清空)

    innobackupex --copy-back /home/mysqlbackup/2021-01-29_11-25-59/

  • 修改数据目录所有者，后面跟上数据目录

    chown -R mysql:mysql /data/mysql5.7/data

mysqldump

  • 新建dump.sh

    #!/bin/bash
mysqldump -h192.168.2.101 -uroot -p***** --single-transaction -C -A --force|mysql -h192.168.2.102 -uroot -p****

  • 更改权限

    chmod 777

  • 运行

    nohup ./dump.sh > dump.log 2>&1 &

    注意：关闭会话时使用exit，否则程序会被关闭

  • 可能出现的错误

    • the user specified as a definer ('***'@'%') does not exits

      • 这是由于迁移视图时该用户没有权限，只需要添加权限即可
      grant all privileges on *.* to xxx@'%' identified by ".";
flush privileges

    • 中断连接

      • 在/etc/my.conf修改配置参数max_allowed_packet大小然后重启
